スパークプラグ： 業務Webアプリケーション向けUIテンプレート集

v2.0.0 WIP

開発技術部に Sparkplug のリクエストメールを送る

UI設計担当のあなたが、Sparkplug を使うべき3つの理由。

HTML/CSS/JSの専門知識は不要。コピペで簡単に高品質なUIが作れます。

かんたん・キレイ

ページテンプレートを選んだら、コンポーネントページから必要なUIパーツのソースをコピペするだけで画面が完成。

ブラウザベースの画面モックを、すぐお客様に提示できます。

ブレない

Sparkplug という基本ルールがあることで、複数のメンバーがUI設計に関わる場合でもレイアウトや見栄えの一貫性を維持できます。

ログイン、検索、編集などのサンプル画面を活用すれば「機能はほぼ同じなのに、作成するメンバーによって見た目が全然違う」といった問題も回避しやすくなります。

実装に使える

Sparkplug は、CSSフレームワークのデファクトスタンダード Bootstrap をベースに開発されています。HTMLやJSのコードはシンプル＆セマンティックなので、画面モックをそのまま実装に利用できます。

また、パーツのレイアウトや見た目をちょっと変更したくなった場合は、Bootstrap の知識をそのまま活用できます。

こんな場面で Sparkplug が活躍。

提案から上流工程にいたるまで、UIに関わるすべての作業が効率アップします。

提案フェーズ

「お客様への営業提案時に綺麗なUIサンプルを提示したいが、作れる人がいない」

Sparkplug を使えば、営業担当のあなた自身がUIサンプルを作成・提案できます。デザインカンプ作成を外部制作会社に依頼する必要はありません。

要件定義フェーズ

「できるだけ早く『紙芝居』をお客様に提示してUIの合意を得たい」「線画だと画面イメージが正確に伝わらない」

Sparkplug ならブラウザで動くリアルな画面モックがすぐ完成。UIの意図がお客様にはっきりと伝わります。
Powerpoint や Excel の線画から“完成予想図”を思い描いてもらう必要はもうありません。

基本設計フェーズ

「お客様からUI設計指針の提出を求められたが、書ける人がいない、書く工数がない」

Sparkplug で提供しているページテンプレートやコンポーネントの振る舞いを定義した「UIデザインガイドライン」を今後公開予定。Sparkplug ベースのUI設計指針を、最小限の手間でお客様と共有できます。

いまどきのメジャーなブラウザに対応。

Windows7 以上または Mac OSX 環境の以下のブラウザで、美しく見栄えの整ったUIを実現します。

Chrome 27+

Firefox 21+

Safari 6+

IE 8+ *

* IE8とIE9ではCSS3プロパティで実現している装飾の一部は省略されます（角丸や文字の周りの影など）
