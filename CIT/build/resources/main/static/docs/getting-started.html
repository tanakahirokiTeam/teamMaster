つかいかた

Sparkplugを使って、ブラウザで動く画面モックを完成させるまでの手順を紹介。

  • 1. Sparkplug を入手
  • 2. ページテンプレートを選ぶ
  • 3. コンポーネントをコピペ
  • 4. リンク指定＆文言調整
  • 5. さらに完成度を高める

1. Sparkplug を入手

開発技術部にメールで最新の Sparkplug パッケージ（Zipファイル）をリクエストします。

開発技術部に Sparkplug のリクエストメールを送る

ボタンを押すとメールソフトが起動します。
起動しない場合は g-uidesign@group.isid.co.jp 宛にメールをお送りください。

担当：荻原（おぎわら）

2. ページテンプレートを選ぶ

Sparkplug のZipファイルは、解凍すると次のような構成になります。
画面モックの作成には、「docs」以外のすべてのフォルダ／ファイルが必要です。

sparkplug-*.*.*/
  ├── css/
  │   └── sparkplug.css
  ├── docs/
  │   └── 略...
  ├── fonts/
  │   └── 略...
  ├── js/
  │   ├── bootstrap.js
  │   ├── jquery-*.*.*.js
  │   └── sparkplug.js
  └── templates/
      ├── _sample_screen_confirm.html
      ├── _template_standard_fixed_1col.html
      └── 略...

「templates」の中にページテンプレート一式が格納されています。
作りたい画面に似たレイアウトのものを1ページ選んでください。

3. コンポーネントをコピペ

ステップ1で選んだページテンプレートを、テキストエディタ（Sublime Text がおすすめ）で開きましょう。

次に、コンポーネントページから、画面に必要なコンポーネント（UIパーツ）を探してください。
たとえば「入力エラー発生中の編集画面」を作りたいなら、入力フォームとしてI/Oボックス、エラーメッセージの表現として「エラーの通知」コンポーネントが必要になるはずです。

コンポーネントが見つかったら、表示サンプルの上にある「HTML」タブをクリック。
サンプルと同内容のHTMLソースが表示されるので、全選択してコピーしてください（ダブルクリックで全選択できます）。

今度はそれをページテンプレート内にペーストしましょう。画面内のコンテンツ表示領域に配置したいなら、以下の領域の中にペーストしてください。

ペーストできたら、ブラウザで表示を確認してみましょう。あとはこのコピペを繰り返して、思い通りの画面を作っていきます。

IE8で表示確認する際の注意

Sparkplug（のベースとなっているBootstrap）では、IE8サポートのためのPolyfillとしてRespond.jsを使用しています。

Respond.jsはローカルのHTMLファイルを開いただけ（URLの頭がfile://になっている状態）では動作しないため、結果としてIE8でレイアウト崩れが発生してしまいます。これを回避するには、画面モック一式をサーバにアップするか、ローカルにサーバ環境を構築する必要があります。

詳しくはRespond.jsのサポートドキュメントを参照してください。

4. リンク指定＆文言調整

リンク指定

画面遷移のトリガーとなるコントロール（ボタンやリンク）に遷移先のhtmlファイル名を指定するだけで、ブラウザで遷移を確認できる、いわゆる「紙芝居」が完成します。

たとえば、編集画面に配置した「上記の内容で登録する」ボタンをクリックすると完了画面（thanks.html）に遷移するようにするには、ボタンのタグ内で「href="#"」となっているところを「href="thanks.html"」のように変更します。「""」で囲まれた部分に、リンク先にしたいHTMLファイル名を書くわけです。

5. さらに完成度を高める

見栄えの変更はテーマを使えばカンタンに行えますが、より見やすく・美しい画面を実現するためには微調整が必要です。デザイナーでなくてもできる、カンタンで効果的なコツをいくつかご紹介します。

データグリッドが綺麗に見えるよう、列幅を個別に指定する

列幅を指定していないデータグリッドは、すべての列が同じ横幅になってしまいます。これは見た目に違和感があるだけでなく、テキストの折り返しの頻発によって可読性が低下する恐れもあるため、特に調整が必要なポイントです。

  • データグリッド内のth要素（テーブルの見出しセル）にstyle属性でwidthを指定します。
  • 自動的に最大幅に広がってほしい列には指定しなくてOK。幅指定しなければ自動的にいっぱいに広がってくれます。
  • テーブルにth要素を使っていない場合は、1行目のtdにwidthを指定してください。

下の例では、左から1,2列目の横幅が固定され、3列目が余った幅いっぱいに伸びて表示されます。

サイドバーの横幅を変更する

サイドバーのデフォルトの横幅は240pxですが、中に入れるリンクテキストの文字数によっては、1文字だけ折り返してしまって読みにくくなる場合があります。その場合は、CSSをいじってサイドバーの横幅を少しだけ広げましょう。

追加＆上書きスタイルはどこに記述するのが正解？

カスタマイズが進むにつれて、プロジェクト独自のスタイルの量も多くなっていきます。
これら独自スタイルをsparkplug.cssに直接追加（または上書き）してしまうのはおすすめできません。

Sparkplug本体のバージョンアップに対応しにくくなる（＝sparkplug.cssの差替えだけでバージョンアップできなくなる）ほか、表示不具合が起きたときの原因切り分けにも手間がかかるためです。

おすすめは、たとえば「overrides.css」といった名前でCSSファイルを新規作成し、そこに独自スタイル一式をまとめる方法です。

htmlでは、sparkplug.cssの後でoverrides.cssが読み込まれるようにします。

「sparkplug.css内のこのスタイルだけを直したい」というときは、sparkplug.cssから該当箇所だけをoverrides.cssにコピーして変更します。スタイルの記述は基本的に後から読み込んだものが優先されるため、html上での読み込み順さえ守っていれば独自スタイルが反映されるはずです。

コンポーネント間の余白を調整する

余白の設定次第で、UIがきちんとした印象になるかどうかが決まります。

  • 上下のコンポーネント間の余白がつまりすぎていて、別の情報・機能なのにひとまとまりに見えてしまう…というときは、Sparkplug の微調整用ヘルパークラスを使って余白を空けましょう。上のコンポーネントに .sp-mb10（下余白を10pxにする）を追加するか、下のコンポーネントに .sp-mt10（上余白を10pxにする）を追加してください。
  • 設定をしても十分な余白が確保できない場合は、上下のコンポーネント間に、余白のための区切りとなる要素 hr.sp-bordered-transparent を追加しましょう。
  • 反対に、コンポーネント間の余白があまりすぎている…という場合は、上下いずれかの余白をゼロにして調整します。上のコンポーネントに .sp-mb0（下余白をゼロにする）を追加するか、下のコンポーネントに .sp-mt0（上余白をゼロにする）を追加してください。

Bootstrap の機能を取り入れる

SparkplugのベースであるCSSフレームワーク Bootstrap には、コンポーネントページで紹介していない便利な要素が多数用意されています。
Sparkplug は Bootstrap の標準コンポーネントと組み合わせても違和感のない見栄えに調整されていますので、複雑なUIを作りたいときは Bootstrap から必要なコンポーネントを取り入れることをおすすめします。

CSS グリッド（レイアウト用カラム）の設定、データグリッドやフォーム、ボタン等、基本となるUI要素のバリエーション。
Components ドロップダウン、ボタンのグループ化、バッジの表現など。
Javascript タブ、リッチなツールチップ、スクロール固定されるナビゲーション（Affix）など。
