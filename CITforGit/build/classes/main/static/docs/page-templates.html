サンプル画面

「ログイン」「一覧」「編集」などといった、業務アプリケーションの典型的な画面に必要なナビゲーションとコンポーネントがあらかじめ配置されたテンプレートです。

手っ取り早く画面を作りたい場合はこちらをベースにするのがおすすめです。サンプルの文言を変更するだけで画面が完成します。コンポーネントを追加／削除することで、より複雑な画面を作ることもできます。

サンプル画面は、横幅固定の左サイドバー付き2カラムレイアウト をテンプレートとして作られています（ログイン画面をのぞく）。

テンプレート

ヘッダー／フッター／サイドバーなどのナビゲーションコンポーネントだけを配置した、中身がカラの画面です。
中身＝「コンポーネント」は、コンポーネントページのサンプルから自分でコピペする必要があります。

「サンプル画面にイメージ通りのものがないので、一から画面を作りたい」という場合はこちらを利用してください。

テンプレートの選びかた

テンプレートが多すぎてどれを選べばいいかわからない…という方に、簡単な選びかたを紹介します。

  • 機能や情報が少なく（今後も増える可能性が低く）、階層の浅いシステム：横幅固定レイアウト1カラム
  • 機能や情報が多いが、階層は浅いシステム：横幅固定レイアウト2カラム｜左サイドバー
  • 機能や情報が多く、階層も深いシステム：横幅可変レイアウト2カラム｜左サイドバー

サイドバーの位置は、特に理由や顧客からの要求がなければ、一般的な慣習に従って「左サイドバー」を使います。
サイドバーを使っていて、かつコンテンツ領域の情報量が非常に多い場合は「横幅可変レイアウト」を使います。
サイドバーを使わない場合や、コンテンツ領域の情報量がそれほど多くない場合は「横幅固定レイアウト」を使うのが良いでしょう。

ちなみに Sparkplug のサイト（このページ）では、横幅固定レイアウト1カラム を使っています。

また、UIの一貫性は、UXを高めるための最も大切な要素です。ログインなどの特殊な画面を除き、システム全体でできるだけ単一のテンプレートを一貫して使うのがおすすめです。画面ごとにナビゲーションの配置を変えてしまうと、ユーザは画面遷移するたびにナビゲーションの機能や意味を理解し直さなければならなくなってしまいます。
