バージョン 1.1.0　2014/07/08 リリース

追加 テンプレートの色合いや見栄えを簡単に変えるためのテーマのスタイルとサンプルを追加しました。
追加 以下のコンポーネントを追加しました。
データグリッドのソート可能な列スタイル .col-sortable .col-sorted
タイトルボックスの透明スタイル .titledbox-transparent
Pillの別スタイル .nav-pills-alt
外周ボーダー付きタブ .tab-bordered .tab-bordered-alt
タブの入れ子 .tab-nested
リストビルダー .table-scrollable
追加 sparkplug.jsにデータグリッドの行選択機能拡張を追加しました。
変更 以下のコンポーネントのスタイルを調整しました。
  • タイトルボックスの一部のボーダーや背景色
  • 文章用スタイル .additional .secondary に下部余白（margin-bottom）を追加
この変更にともない、v1.0.4以前に作成した画面モックの一部で表示が変化する場合があります。その場合、次のように対応してください。
  • 文章用スタイルの下部に不要な余白が発生している場合は、要素に .mb0 を追加して調整してください。
  • input, textarea, selectなどのフォーム要素が縦に並んでいる箇所で要素同士がくっついて見える（要素間の上下の余白がゼロになっている）場合、下の要素に .mt10 を追加して調整してください。
変更 v1.0.4以前で、データグリッドを単独で使う際のボーダー表示調整に使われていた .table-front, .table-rearクラスを削除し、クラス未指定でも自動的に見栄えが整うよう調整しました。
今後 .table-front, .table-rear の記述は不要ですが、（v1.0.4以前に作成した画面モックなどで）html上にクラス指定が残っていても、表示上の支障はありません。
変更 サンプル画面の構成と内容を見直しました。
  • サンプル画面（一覧）の配置コンポーネントを拡充
  • サンプル画面（編集）の配置コンポーネントを拡充
  • サンプル画面から「サブヘッダー常時固定版」を削除
変更 Bootstrapを v2.2.2 から v2.3.2（2系の最終バージョン） にバージョンアップしました。
変更 Font-Awesomeを v3.0.2 から v3.2.1（3系の最終バージョン） にバージョンアップしました。
変更 jQueryを v1.10.2 から v1.11.1（1系の最新バージョン） にバージョンアップしました。
修正 IE8-9がautofocus/placeholder属性に対応していない問題をPolyfill（jquery.placeholder.js）の導入で解消しました。
修正 IE8において、データグリッド最終行のtd下端にボーダーが表示されない問題を解消しました。

バージョン 1.0.3　2013/08/16 リリース

追加 行選択データグリッドに、一括実行アクションエリア（選択された行に対する一括実行アクションボタンを配置するための領域）を追加しました。
追加 Sparkplug の利用規約を追加しました。これは Sparkplug の提供を受けたお客さま・協力会社のみなさまに向けたもので、Sparkplugを使用するには本規約への同意が必須となります。
修正 変更 コンポーネントのHTMLソース表示等に使用しているシンタックスハイライト用JSプラグインを、SyntaxHighlighter から CodeMirror に変更しました。これにより、HTMLソースのペースト時にスタイル崩れが起きる問題を解消しました。
修正 アイコン付きのエラーメッセージ（.list-with-icon）をより読みやすくするため、2行目以降に付けられているインデントを調整しました。
修正 アイコンボタン（.btn-icon-*）の表示を改善しました。

バージョン 1.0.2　2013/07/17 リリース

変更 Sparkplugダウンロードパッケージ（Zipファイル）の構成を変更し、Sparkplugを使うための情報をdocsフォルダに分離しました。
画面モックの作成には、docsフォルダを除いたフォルダ／ファイル一式をお使いください。
変更 コンポーネントのHTMLソース表示等に使用しているシンタックスハイライト用JSプラグインを、Google Code Prettify から SyntaxHighlighter に変更しました。これにより、IE8,9以外のブラウザでは、ダブルクリックでHTMLソースを全選択できるようになりました。
変更 すべてのドキュメントの改行コードをLFからCR+LFに変更しました。
修正 サブヘッダー常時固定版／横幅可変／1カラムのページテンプレート（_template_fixed_sp-subnav_fluid_1col.html）において、コンテンツエリア上部に不必要な余白が入っていた問題を修正しました。

修正 テンプレートとその他のページで別々のjsファイルを読み込んでいた（テンプレートでは結果的にjsファイルの読み込みに失敗していた）問題を修正し、全ページで非minify版のjsファイルを読み込むよう統一しました。
修正 「ページの先頭へ」ボタンが .fixed-sidenav の下に隠れてしまう問題を修正しました。

先行配布バージョンからの変更点

修正 変更 Webkit以外のブラウザでI/Oボックスに.skeletonクラスを指定した際、行マージしたセルの右側にあるセルの一部で左ボーダーが表示されなくなる問題を修正しました。
これにともない、.skeletonクラス指定時の表示を次のように変更しました。
.iobox-hまたは.iobox-vとの組み合わせによっては不要な縦ボーダー／横ボーダーが表示される場合があるため、確認が必要です。
  • 新：.iobox-hで使うとテーブルの上下端をのぞく横方向の区切り線を表示、.iobox-vで使うとテーブルの左右端を含む縦方向の区切り線を表示

変更 ページテンプレートの構成をシンプルにするため、使い勝手の悪い「Bootstrap標準の.span*で幅指定したサイドバー」付きテンプレート計8ページを削除しました。

  • _template_standard_fixed_2col_r.html

修正 サンプル画面「ポータル」でのタイトルボックスの表示不具合を修正しました。
修正 IEにおけるヘッダーの.brandテキストの表現を向上しました。
修正 IEで入力コントロールおよびselect要素のラベルがメイリオで表示されない問題を修正しました。
修正 IE8,9でタイトルボックスの「タイトルが白」スタイルが適用されない問題を修正しました。

修正 IE8でデフォルトボタンと.wellの背景色が似通っていて見にくい問題を修正しました。

修正 IE8で、Bootstrap標準の.span*で幅指定した入力コントロールが巨大化する問題を修正しました。

既知の問題

未解決 Firefoxにおいて、「templates」フォルダ内のhtmlファイルをローカル環境で開いた場合、アイコンフォント（FontAwesome）が正しく表示されない。画面モックの表示にFirefoxを使う場合は、この問題が解決するまで次のいずれかの方法で対処することをおすすめします。
  • 「templates」フォルダ内のhtmlファイルを、すべて「sparkplug-*.*.*」の直下に移動したうえで、htmlファイル内の（CSSやJSなどの）外部ファイル読み込み指定箇所を適切なパスに変更する
  • 画面モック一式をWebサーバ上に置く

追加予定の機能

追加予定 特定の条件で特定のコントロールを自動的にディスエーブル／イネーブルする機能。
たとえば、あるフォーム内の必須入力項目がひとつでも未入力のときは実行ボタンをディスエーブル表示し、すべて入力されると自動的にイネーブルに変化するようなもの。
